The goal is to re arrange a 3x3 grid of numbered tiles from an initial state to a goal state using the minimum number of moves. This report presents the implementation and performance analysis of three search algorithms: Depth-First Search (DFS), Breadth-First Search (BFS), and A* .</p><p style="text-indent: 0pt;text-align: left;"><br/></p><p class="s4" style="padding-top: 13pt;padding-left: 5pt;text-indent: 0pt;text-align: left;">Assumptions</p><p style="text-indent: 0pt;text-align: left;"><br/></p><ul id="l2"><li data-list-text="●"><p class="s5" style="padding-left: 41pt;text-indent: -18pt;line-height: 114%;text-align: left;">The initial state of the puzzle is entered by the user, and it consists of 9 unique numbers from 0 to 8, where 0 represents the empty tile.</p><p style="text-indent: 0pt;text-align: left;"><br/></p></li><li data-list-text="●"><p class="s5" style="padding-left: 41pt;text-indent: -18pt;line-height: 114%;text-align: left;">The goal state is a predefined arrangement where numbers are in ascending order from left to right, top to bottom.</p></li></ul><h3 style="padding-top: 7pt;padding-left: 64pt;text-indent: 0pt;text-align: center;">Data Structure Used</h3><p style="text-indent: 0pt;text-align: left;"><br/></p><p class="s6" style="padding-left: 5pt;text-indent: 0pt;text-align: left;">Puzzle State Representation</p><p style="padding-top: 15pt;padding-left: 5pt;text-indent: 0pt;line-height: 114%;text-align: left;">The puzzle state is represented as a 3x3 matrix, where each element represents a tile in the puzzle.</p><p style="padding-top: 12pt;padding-left: 5pt;text-indent: 0pt;line-height: 114%;text-align: left;">The “PuzzleState” struct is utilized to encapsulate the puzzle state as a 2D Vector, the “zero index” position.</p><p style="padding-top: 12pt;padding-left: 5pt;text-indent: 0pt;line-height: 114%;text-align: left;">The “Node” struct is utilized to encapsulate the puzzle state as an instance of a “PuzzleState” class, the “parent” node, the action direction.</p><p style="text-indent: 0pt;text-align: left;"><br/></p><p class="s6" style="padding-left: 5pt;text-indent: 0pt;text-align: left;">Node Expansion &amp; Exploration</p><p style="padding-top: 15pt;padding-left: 5pt;text-indent: 0pt;line-height: 114%;text-align: left;">In <u>DFS &amp; BFS</u> To expand the nodes we used a certain method, in which we declared two vectors that store the values of the possible index moves and then iterated on every possible move such as left, right, up, down.. if it is a valid move we add it to our frontier container and use it, checking if it is our goal or it can be our next parent node.</p><p style="text-indent: 0pt;text-align: left;"><br/></p><p style="padding-top: 4pt;padding-left: 5pt;text-indent: 0pt;line-height: 114%;text-align: left;">In <u>A*</u> When we call the function “solveA” we have our current node, in a while loop we check if the state of the goal is not our goal, then we start expanding the childes of these node by: first checking if every move such as Up, Down, Right and Left is a valid move or not by checking the position of the zero value “empty tile<span class="s8">” </span>that is stored in the variables x &amp; y, if the move is valid we add it to our frontier of the type priority queue and then choosing the node with the least F and then add it to the visited list, at the end we update our current node to the chosen node of the least f using manhattan or euclidean distance (choosen by the user).</p><p style="text-indent: 0pt;text-align: left;"><br/></p><p class="s6" style="padding-left: 5pt;text-indent: 0pt;text-align: left;">Set of the Explored</p><p style="padding-top: 15pt;padding-left: 5pt;text-indent: 0pt;line-height: 114%;text-align: left;">To store every node state that we already used, we convert our state to a string using a “toString” function and then store it in an unordered set of states in a “string” shape, which will allow us to easily access and search on the set to prevent visiting any explored node again.</p><p class="s6" style="padding-top: 3pt;padding-left: 5pt;text-indent: 0pt;text-align: left;">Search Algorithm</p><p style="padding-top: 15pt;padding-left: 5pt;text-indent: 0pt;line-height: 114%;text-align: left;">A class called “Search Algorithm” is used to determine some common methods the algorithms need such as “isGoalState” function to compare our current state to the goal, “toString” to convert the state to a string, “printPuzzle” to print the puzzle state in each iteration, each algorithm class can access this class.</p><p style="text-indent: 0pt;text-align: left;"><br/></p><h4 style="padding-left:
